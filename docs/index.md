# Sentence-Citation RAG Prototype
{: .fs-9 }

A prototype RAG system that gives tax auditors **sentence-level citations**
they can verify before signing off on AI-assisted work.

## The problem

Tax auditing is **high-stakes sign-off work**. An auditor's name goes on
the final determination, and if the determination is wrong there are real
regulatory and financial consequences. This is why auditors have been
slow to adopt AI assistants — not because the assistants can't produce
reasonable-sounding answers, but because there is no efficient way for
the auditor to verify an answer without re-doing the research by hand.

The prototype is built on a simple hypothesis:

> If an AI answer comes with a citation for **every sentence** — pointing
> at the exact source sentence that supports it — an auditor can verify
> at roughly the speed of reading, and confidence in the tool grows fast
> enough to unlock more ambitious agentic workflows later.

## What Phase 1a proves

- **A working pipeline:** ingest → chunk → dual-layout index →
  retrieve → generate → cite at the sentence level.
- **Two citation strategies** compared head-to-head:
  - **Inline prompted** — the answer model is instructed to emit citation
    markers as it writes.
  - **Post-generation alignment** — the answer is written first, then a
    separate aligner matches each sentence to its best-supporting source.
- **A self-serve evaluation harness** that measures
  precision / recall / F1, coverage, retrieval recall, judge-rated
  **faithfulness**, and **citation stability** across repeated runs.
- **A public-benchmark adapter** (HAGRID / ALCE) for apples-to-apples
  comparison against published attribution numbers.
- **A layperson review workflow** — ML-Eng / PM can spot-check ground
  truth for obvious breakage and tag each item with reviewer
  confidence. Clearly labelled **non-SME** everywhere it surfaces.

See [Phase 1a overview]({{ '/phase-1a/overview/' | relative_url }})
for the full walkthrough in plain language, and

## What Phase 1a deliberately did NOT do

- **No SME validation** of ground truth. Phase 1a uses synthetic ground
  truth generated by one model and judged by a different model. That
  is enough to debug plumbing and get comparative numbers, but the
  headline accuracy claims to a customer need SME sign-off —
  which is Phase 1b work.
- **No agentic autonomy.** Every component is single-shot
  retrieve-then-generate; no planning loops, no tool use. The point of
  Phase 1a is to earn the right to do that next.

## Repository layout

```
src/sentcite/         core pipeline modules
  ingest/             PDF → parsed JSON (Azure Document Intelligence)
  chunking.py         parsed JSON → chunks + sentences
  indexing/           Azure AI Search dual-layout indexer
  retrieval.py        hybrid search
  generate.py         answer generation (2 strategies)
  cite_align.py       post-gen alignment citation
  judge.py            LLM-as-judge faithfulness scoring
  self_consistency.py citation-set stability across replicas
  eval.py             full evaluation harness
  benchmarks.py       HAGRID / ALCE adapter
  layperson_review.py non-SME spot-check workflow
scripts/              CLI entry points
tests/                175 tests, all passing
notebooks/            walkthrough notebooks
docs/                 this site
research/             Copilot Researcher Agent outputs
```
